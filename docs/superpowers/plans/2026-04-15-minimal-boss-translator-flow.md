# 老板语言翻译器极简交互开发计划

## 目标

把当前首页改造成单页直出的极简交互，只保留一个主动作：`上传老板聊天截图`，并围绕“先给最佳回复，再让用户按需看解释/切换风格/修改 OCR/解锁高级包”来重构整个主链路。

## 已锁定的交互骨架

- 单页，不跳结果页
- 首屏只有一个主按钮
- `直接粘贴文本` 改成底部弹层
- 结果区默认只突出一条最推荐回复
- OCR 校对默认隐藏，点击后原地展开
- 高级包只在结果页底部轻露出
- `先看示例` 保留为弱入口
- 本轮先不做历史记录入口和历史页

## 前端改造清单

### 1. 页面状态机
需要把首页收敛成以下状态：

- `idle`
- `analyzing`
- `result`
- `editing`
- `error`
- `premium_unlocked`

### 2. 页面布局
首页只保留：

- 一句价值主张
- 一个主按钮：`上传老板聊天截图`
- 两个弱入口：`直接粘贴文本`、`先看示例`

结果页固定顺序：

1. 一句潜台词总结
2. 两个轻量标签
3. 一张主回复卡
4. 一行轻量 tab
5. OCR 修改入口
6. 高级包入口 / 高级包内容

### 3. 默认推荐回复
前端不再默认写死展示某一轨，而是优先使用后端返回的 `recommendedTrackId`。

如果后端没返回，则回退到：

- `align`
- 没有 `align` 时，再取第一条可用轨道

### 4. OCR 编辑
只有点击 `识别有误？修改文本` 时才展开文本框。展开区只保留：

- 文本框
- `重新解码` 按钮

### 5. 高级包
高级包只允许追加到结果页底部，不刷新整页，不重排基础结果。

## 云函数改造清单

### 1. 输出结构
补充：

- `recommendedTrackId`

继续保留：

- `ocrText`
- `ocrMeta`
- `analysis`
- `tracks`
- `premium`
- `teaser`

### 2. 规则兜底
即便未接 LLM，也要保证能返回：

- 一条合理的默认推荐轨道
- 三轨完整回复
- 高级包演示结果

### 3. 响应归一
对模型输出做一层清洗：

- 过滤非法 `track.id`
- 补齐缺失字段
- `recommendedTrackId` 缺失时按规则回退
- 超范围场景优先 `reject`

## 验证点

### 体验验证
- 新用户进入后 3 秒内能理解产品用途
- 首屏只感知到 1 个主动作
- 上传 / 粘贴后先看到的是“建议你直接回”
- 用户不理解三轨也能完成第一次复制

### 结构验证
- 页面不出现旧版大而全介绍模块
- OCR 文本框不是常驻的
- 高级包入口不抢首屏焦点
- 高级包解锁后只做局部追加

### 技术验证
- `recommendedTrackId` 可以稳定驱动默认展示轨道
- 规则兜底可以在无 LLM 时跑通全链路
- premium 解锁不会破坏基础结果状态
